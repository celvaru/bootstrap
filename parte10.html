📌 Sass en Bootstrap: Variables y personalización básica

¿Qué es Sass y por qué usarlo con Bootstrap?

Sass (Syntactically Awesome Stylesheets) es un preprocesador CSS que permite usar variables, anidación, mixins y funciones. Bootstrap 5 está construido con Sass, lo que facilita la personalización de temas, colores, tipografías y espaciados sin tener que modificar archivos fuente directamente. Sobreescribiendo variables de Sass podemos cambiar la apariencia global de Bootstrap de forma limpia y mantenible.

✨ Ejemplos de personalización con Sass
🎨 1. Sobreescritura de colores primarios

Variables como $primary, $purple se redefinen antes de importar Bootstrap.

✅ Este alerta usa el color primario personalizado vía Sass.
$primary: #6f42c1; $purple: #b980ff; $body-bg: #f5f0ff;
🔤 2. Tipografía (Font family y tamaños)

Variables: $font-family-sans-serif, $font-size-base, $headings-font-weight

Texto normal con peso de fuente headings: 600 y énfasis. Los espaciados también se ajustan con variables como $headings-margin-bottom.

$font-family-sans-serif: 'Poppins', system-ui; $font-size-base: 1.125rem; $headings-font-weight: 600;
📏 3. Espaciados (Margin y Padding personalizados)

Sass permite modificar $spacer y sus multiplicadores ($spacers map).

margin-bottom: 1.5rem (customizado)
padding: 2rem (variable $spacer * 2)
padding base $spacer: 1rem
$spacer: 1rem; $spacers: ( 0:0, 1:0.5rem, 2:1rem, 3:1.5rem, 4:2rem, 5:3rem );
⚙️ 4. Componentes personalizados (Botones)

Modificando $btn-border-radius, $btn-padding-y y colores.

📌 $btn-border-radius: 50rem; | $btn-padding-y: 0.5rem;
📄 Código Sass: custom.scss

Así se sobreescriben variables antes de importar Bootstrap:

💡 ¿Cómo compilar? Con npm run build o usando Dart Sass: sass custom.scss custom.css. El CSS resultante integra todos los cambios.
🔍 Explicación visual: Mapa de variables en Bootstrap
  • ✔️ Variables predeterminadas: _variables.scss → 900+ opciones.
  • ✔️ Sobreescritura: Declarar tus variables antes del @import "bootstrap".
  • ✔️ Personalización avanzada: Modificar maps como $theme-colors, $grid-breakpoints.
  • ✔️ Ventajas: Código mantenible, sin modificar archivos de Bootstrap, fácil actualización.

Flujo: variables.scss → custom.scss → bootstrap → estilos finales
